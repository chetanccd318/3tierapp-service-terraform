## 🟦 Project Title

**3-Tier Architecture Deployment on AWS using Terraform (Infrastructure as Code)**

---

## 🟩 Project Description

This project automates the deployment of a scalable 3-tier application architecture on Amazon Web Services using Terraform. It provisions 15+ AWS services including networking, compute, and storage resources entirely through Terraform CLI, following Infrastructure as Code (IaC) principles to ensure consistent, repeatable, and environment-independent deployments across dev, staging, and production environments.

---

## 🟨 Step-by-Step Implementation (Terraform CLI Based)

### 1. Initialize Terraform Project

```bash
terraform init
```

* Initializes working directory
* Downloads required provider plugins

---

### 2. Validate Configuration

```bash
terraform validate
```

* Checks syntax and configuration correctness

---

### 3. Review Execution Plan

```bash
terraform plan
```

* Previews infrastructure changes before deployment

---

### 4. Deploy Infrastructure

```bash
terraform apply -auto-approve
```

* Provisions complete AWS infrastructure automatically using IaC

---

### 5. Destroy Infrastructure (Optional)

```bash
terraform destroy -auto-approve
```

* Removes all created AWS resources when not needed

---

## 🟧 Key Implementation Highlights

* Fully automated infrastructure provisioning using Terraform CLI (no manual AWS Console steps)
* Modular Terraform code structure for reusability and scalability
* Supports multiple environments: development, staging, and production
* Implements Infrastructure as Code (IaC) best practices for DevOps workflows
* Provisioned 15+ AWS services including compute, networking, and storage
